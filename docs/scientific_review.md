# 科学性 Review

## 目的

本文用于明确当前平台各评分与模型模块的科学边界，回答三件事：

1. 哪些部分具有明确的机理或文献方向支撑。
2. 哪些部分仍属于工程启发式设计。
3. 哪些输出适合做相对排序，哪些不适合被解释为绝对真值。

## 总体结论

当前平台更适合被定义为：

- 可解释的相对风险排序平台
- 机制约束下的工程化竞赛分析系统

当前平台不应被表述为：

- 船体真实污损厚度或真实生物量的直接观测模型
- 工业级正式碳核算或正式燃油结算模型
- 区域真实污损分布的直接测绘产品

## FPI

### 具有机制支撑的部分

- 行为暴露是污损积累倾向的主因，这与 IMO biofouling 风险逻辑一致。
- 海温、盐度、生产力代理和水动力条件会共同影响附着生物的存活、繁殖和早期附着窗口。
- 维护状态不应完全脱离污损积累判断，因为维护历史会改变相同暴露下的风险水平。

### 仍属启发式设计的部分

- 海温、盐度、叶绿素和流速的具体分段切点，不是某篇文献原文给出的统一真阈值，而是“文献定方向、热带港区定尺度”的工程落地。
- `EnvModifier = 0.40T + 0.20S + 0.25P + 0.15H` 是工程权重分配，不是文献直接给出的固定系数。
- `FPI = BehaviorExposure × EnvAdj × MaintenanceAdj` 是机制优先的工程结构，用于避免环境项单独抬分造成的假阳性。

### 适用边界

- 适合：单船相对风险排序、维护优先级筛查、同批对象比较。
- 不适合：绝对污损厚度判断、不同海区间的无校准横向比较、物种级生态结论。

## ECP

### 具有机制支撑的部分

- 船体污损会增加阻力，并进一步抬高燃油和排放代价，这一方向具有明确工程常识与航运研究支撑。
- 将 ECP 放在 FPI 之后，作为潜在代价的放大器，结构上比“另做一套第二版 FPI”更合理。

### 仍属启发式设计的部分

- 当前 `CarbonPenaltyModifier` 仍是代理结构，不是阻力试验模型，也不是正式碳核算模型。
- 其中维护负担与持续暴露的系数，属于工程参数，用于表达“污损一旦形成，潜在代价会被进一步放大”。

### 适用边界

- 适合：相对能耗惩罚表达、维护优先级排序、风险展示。
- 不适合：正式碳核算、真实燃油结算、运营财务结算。

## RRI

### 具有机制支撑的部分

- 区域热点确实应由环境适宜性、交通密度、停留概率和港口/锚地暴露共同表达。
- 区域层更接近“促进污损积累的空间条件”，不等于区域真实污损现状图。

### 仍属启发式设计的部分

- 当前格网尺度、组合权重和风险等级，属于平台表达层设计。
- `StayProb` 与 `PortAnchorageIntensity` 的拆分，是为减少重复计分的工程化处理。

### 适用边界

- 适合：区域热点识别、巡查优先级判断、空间对比。
- 不适合：真实污损分布测绘、生态调查替代品、所谓“真实污损地图”。

## 异常检测

### 具有方法支撑的部分

- 无监督异常检测适合回答“哪些对象明显偏离常规暴露模式”。
- Autoencoder 用重建误差识别偏离样本分布的对象，这一方法逻辑成立。

### 仍属启发式设计的部分

- 当前异常类型识别仍是“异常分数 + 规则归类”的混合结构，不是独立训练的监督式分类器。
- `highly_abnormal / suspicious / normal / observation_insufficient` 的阈值与标签属于工程分层。

### 适用边界

- 适合：总览页筛查、异常对象提示、单船解释增强。
- 不适合：直接证明“已经发生严重污损”。

## 时序预测

### 具有方法支撑的部分

- 用历史 AIS 行为与环境暴露序列预测下一时间窗风险，属于合理的短期时序建模方向。
- LSTM 作为轻量时序模型，适合当前竞赛规模和数据量。

### 仍属启发式设计的部分

- 当前预测目标是 `FPI proxy`，不是外部观测到的真实污损值。
- 风险等级阈值来自验证集校准，是平台内部口径，不是行业标准。

### 适用边界

- 适合：短期趋势预警、下一窗口风险提示。
- 不适合：长期生态演化预测、真实污损厚度预测。

## 当前最稳妥的表述口径

- `FPI / ECP / RRI` 是可解释代理指标，不是绝对真值。
- 环境因子的作用方向由机理和文献支撑，但具体切点和权重仍是热带港区场景下的工程先验。
- 异常检测回答的是“偏离常规模式的程度”，不是“真实污损已经发生”的直接证明。
- LSTM 回答的是“短期趋势”，不是长期生态演化。

## 当前仍待加强的部分

- 更细粒度的文献化参数校准。
- 更丰富的外部校验样本。
- 更强的跨窗口、跨区域稳定性验证。
- ECP 与真实能耗/排放代价之间的外部映射验证。
