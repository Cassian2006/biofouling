# 科学性 Review

## 目的

本文件用于明确当前平台各个评分与模型模块的科学性边界，回答三件事：

1. 哪些部分具有明确的文献或机制支撑
2. 哪些部分属于工程启发式设计
3. 哪些输出只适合相对排序，不适合绝对判断

## 总体结论

当前平台更适合被定义为：

- `可解释的相对风险排序平台`
- `机制约束 + 工程实现` 的竞赛级分析系统

而不是：

- 直接估计真实船底污损厚度的生态模型
- 直接输出工业级绝对燃油惩罚或绝对碳排惩罚的运营系统

## 模块逐项审查

### 1. FPI

#### 有文献或机理支撑的部分

- 行为暴露是主因，而环境因子是调制项，这与 [IMO Biofouling](https://www.imo.org/en/OurWork/Environment/Pages/Biofouling.aspx) 的风险管理逻辑一致。
- 温度、盐度、生产力和局地流动条件确实会影响附着生物的存活、变态、繁殖和早期附着窗口。
- 藤壶 cyprid 临时附着会受到局地流速影响，高流速条件通常更不利于稳定附着，方向上可参考 [Journal of Experimental Biology: Temporary adhesion of barnacle cyprids](https://journals.biologists.com/jeb/article/211/13/2178/18261/Temporary-adhesion-of-barnacle-cyprids-effects-of/18261).

#### 工程启发式设计

- `sst / salinity / chlorophyll-a / current_speed` 的具体分段切点不是某篇文献直接给出的统一真阈值，而是：
  - 机制方向由文献支持
  - 热带港区场景由研究区先验约束
  - 局地尺度由当前数据分布辅助校准
- `EnvModifier = 0.40T + 0.20S + 0.25P + 0.15H` 是工程落地权重，不是原始文献直接给出的固定系数。
- `FPI = BehaviorExposure × (0.7 + 0.3 × EnvModifier)` 是机制优先的设计选择，用于避免“环境过强、行为过弱”导致的假阳性。

#### 适用边界

- 适合：当前窗口内的相对风险排序、单船暴露诊断、同批对象比较
- 不适合：跨海区直接比较、绝对污损厚度判断、物种级生态结论

### 2. ECP

#### 有文献或机理支撑的部分

- 船体污损会增加阻力、带来燃油与排放代价，这一方向具有明确工程与海洋运维文献基础。
- 在平台里，把 ECP 作为 FPI 的后续惩罚量而不是独立风险量，是合理的结构选择。

#### 工程启发式设计

- 当前 ECP 仍不是工业级燃油模型。
- 其放大系数依然主要是工程惩罚项，用于表达“风险上升后，潜在能耗代价可能同步放大”。
- 因此 ECP 目前只能解释为 `relative penalty proxy`，不应解读为真实燃油百分比或真实吨碳排。

#### 适用边界

- 适合：辅助维护优先级判断、展示“风险升高可能带来的代价放大”
- 不适合：真实燃油成本结算、正式碳核算

### 3. RRI

#### 有文献或机理支撑的部分

- 将空间热点表示为环境条件、船流强度、低速停留和锚地压力的组合，是合理的港区风险表达。
- RRI 更接近“区域促进条件”而不是“真实污染结果”，这一定位是清晰且科学上稳妥的。

#### 工程启发式设计

- 当前网格尺度、组合权重和风险分级仍是平台级表达设计。
- 研究区换到别的海域后，需要重新校准环境权重与交通权重。

#### 适用边界

- 适合：热点识别、区域比较、优先巡查
- 不适合：绝对生态强度判断

### 4. 异常检测

#### 有文献或机理支撑的部分

- 使用无监督异常检测去发现“明显偏离常规暴露模式”的对象，方法上是成立的。
- Autoencoder 的含义是学习常规模式，再用重建误差识别偏离对象，这与当前任务匹配。

#### 工程启发式设计

- 当前异常类型识别仍是 `Autoencoder 分数 + 规则归类`，不是单独训练的监督分类器。
- `highly_abnormal / suspicious / normal / observation_insufficient` 的切点是工程分级规则。

#### 适用边界

- 适合：筛查异常个体、支持总览页与单船页解释
- 不适合：直接认定“已发生真实严重污损”

### 5. 时序预测

#### 有文献或机理支撑的部分

- 使用历史 AIS 行为与环境暴露序列预测下一时间窗风险，是合理的时序建模方向。
- LSTM 对短历史窗口风险预测是可解释且工程上成熟的轻量选择。

#### 工程启发式设计

- 当前模型目标是预测 `FPI proxy`，而不是预测真实污损观测值。
- 风险等级阈值由验证集校准得到，因此是平台内标签，不是外部标准。

#### 适用边界

- 适合：短期趋势预测、风险预警
- 不适合：长期生态演化判断、绝对生物量预测

## 当前最重要的科学表达口径

评审或答辩时，建议统一使用以下口径：

- `FPI / ECP / RRI` 是可解释代理指标，不是绝对真值
- 环境因子的方向由机制和文献支持，具体切点和权重是热带港区场景下的工程先验
- 异常检测与预测模块增强的是 `筛查与预警能力`，不是替代规则链路的黑箱主引擎

## 当前仍待继续加强的部分

- 更细粒度的文献化参数校准
- 不同时间窗与不同海区的迁移验证
- 更真实的外部校验样本
- 对 ECP 的燃油与碳代价映射校准
