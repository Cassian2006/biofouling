# AIS 下载接入说明

## 现状

本机目录 [C:\Users\cai yuan qi\Desktop\下载AIS数据](/c:/Users/cai%20yuan%20qi/Desktop/%E4%B8%8B%E8%BD%BDAIS%E6%95%B0%E6%8D%AE) 中已经存在一套 AIS 下载相关脚本，主要分为三步：

1. 查询研究区内船舶进出或穿越信息
2. 从区域结果中提取 MMSI
3. 根据 MMSI 拉取 AIS 轨迹

## 当前判断

这套脚本可以复用思路，但不建议原样直接并入当前仓库。原因有三个：

- 存在硬编码绝对路径
- 存在硬编码时间范围
- 存在硬编码鉴权信息

## 建议接入方式

### 第一阶段

- 不复制旧脚本源码到当前仓库
- 先把它们视为“外部下载工具”
- 只要求其最终导出 CSV 或 JSON 结果
- 再由当前仓库的清洗脚本接管后续处理

### 第二阶段

如果后续确认该数据源稳定可用，再做正式集成：

1. 把鉴权信息改为环境变量读取
2. 把时间范围、研究区、输出目录改为命令行参数
3. 把下载结果统一落到当前仓库 `data/raw/`
4. 再与 `scripts/ingest_ais.py` 串成同一条流水线

## 当前最稳妥方案

在真实下载阶段，建议先用现有外部脚本拿到一份 AIS 文件，然后放入当前项目的 `data/raw/`，用本项目的清洗和特征流程继续处理。

这样做的优点是：

- 不会把旧脚本中的路径与密钥耦合带入新项目
- 不会让当前仓库过早绑定某个外部数据接口
- 能先完成本项目自己的数据标准化与分析链路
