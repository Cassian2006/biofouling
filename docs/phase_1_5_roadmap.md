# 1.5 路线草案

## 定位

`1.0` 的目标是完成一个完整、清洗、强体验、强可视化展示的平台。

`1.5` 的目标不是推翻当前规则链路，而是在 `1.0` 已稳定的基础上，补两类轻量深度学习模块：

- 一类回答“未来会怎样”
- 一类回答“谁明显不正常”

这样可以让平台从“当前状态诊断”升级为“趋势预测 + 异常发现”的组合能力。

## 模块一：FPI 时序预测

### 目标

基于船舶过去一段时间的 `AIS 行为序列` 与 `海洋环境暴露序列`，预测其下一时间窗的：

- `FPI` 数值
- 或 `风险等级`

### 输入

- 航速、低速比例、停留时长、锚泊时长等时间序列
- `sst / chlorophyll_a / salinity / current_u / current_v` 暴露序列
- 可选的港口访问节律与时间窗统计量

### 模型方向

首版建议使用轻量时序模型：

- `LSTM`
- 或后续可对比更轻的 `GRU`

### 输出

- 下一时间窗 `FPI_pred`
- 或 `low / medium / high` 风险等级预测

### 平台作用

- 把当前平台从“规则诊断”推进到“未来趋势预测”
- 支持单船页的风险走势前瞻
- 支持总览页的未来高风险对象预警

### 约束

- 1.5 阶段仍以轻量、可解释、可部署为原则
- 首版不追求复杂大模型，不引入重训练基础设施

## 模块二：异常污损暴露检测

### 目标

基于船舶当前的 `行为特征` 与 `环境暴露特征`，识别与同类船相比暴露模式明显异常的个体。

### 输入

- 当前单船特征向量
- 同期同区域船舶样本
- 可选的船型、活动区域、停留模式分组

### 模型方向

首版建议使用轻量无监督模型：

- `Autoencoder`

也可保留与传统异常检测方法对比的空间，但首版主线先保持统一。

### 输出

- `anomaly_score`
- `normal / suspicious / highly abnormal`

### 平台作用

- 快速发现“异常高风险船舶”
- 提升总览页重点船舶筛查效率
- 提升区域页热点与异常个体联动解释能力

## 与 1.0 的关系

这两个模块都建立在 `1.0` 现有资产之上：

- 已有 AIS 清洗链
- 已有环境暴露链
- 已有单船特征表
- 已有 FPI / ECP / RRI 规则基线

因此 `1.5` 应该采用“增量叠加”方式，而不是重做主链路。

## 推荐推进顺序

1. 先做 `FPI 时序预测`
2. 再做 `异常污损暴露检测`

原因：

- 时序预测与当前单船分析链衔接最自然
- 它更容易直接增强单船页与总览页的核心叙事
- 异常检测更适合作为第二层筛查与预警增强

## 收官判断

如果 `1.0` 收尾完成，`1.5` 的第一阶段就应聚焦以下成果：

- 形成可训练的数据集切片方案
- 产出首版轻量模型训练脚本
- 将预测结果或异常分数接入现有前后端页面
- 保持规则基线与学习模型并行展示，避免黑箱替代
