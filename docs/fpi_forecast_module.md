# FPI 时序预测模块起步说明

## 当前阶段

这是 `1.5` 的第一步落地版本，目标是先把：

- 时序训练样本
- 轻量模型入口

搭起来，而不是立刻把预测结果接进前端页面。

## 已落地内容

### 1. 序列特征构建模块

文件：

- `scripts/fpi_forecast.py`

作用：

- 将清洗后的 AIS 与环境数据按固定时间窗聚合
- 生成窗口级特征
- 构造监督式序列样本

当前默认输入逻辑：

- `6 小时` 一个窗口
- 取历史 `8` 个窗口作为输入
- 预测下一 `1` 个窗口的 `FPI proxy`

### 2. 数据集导出脚本

文件：

- `scripts/build_fpi_sequence_dataset.py`

作用：

- 直接从当前处理后的 AIS 与环境 CSV 生成时序预测训练数据集

当前已在真实 15 天窗口上验证：

- 窗口行数：`5495`
- 序列样本数：`1160`

### 3. LSTM 训练脚本

文件：

- `scripts/train_fpi_lstm.py`

作用：

- 读取扁平化后的序列数据集
- 按船舶切分训练集与验证集
- 训练首版轻量 `LSTM` 回归模型
- 输出模型权重与训练指标

## 依赖说明

当前仓库默认依赖中还未内置 PyTorch。

原因：

- 1.0 阶段主链路不需要深度学习依赖
- 1.5 当前刚启动，先把数据集和训练入口做好更稳妥

因此：

- `scripts/build_fpi_sequence_dataset.py` 可直接运行
- `scripts/train_fpi_lstm.py` 运行前需要单独安装 PyTorch

## 首版训练结果

当前已在真实 15 天样本上完成首版 `LSTM` 训练：

- 训练样本：`926`
- 验证样本：`234`
- 隐层大小：`32`
- 层数：`1`
- 训练轮数：`20`

当前最佳验证结果：

- `best_val_loss = 0.044407`
- `best_val_rmse = 0.21073`

模型产物输出目录：

- `outputs/models/fpi_lstm_20260115_20260130/`

当前判断：

- 这已经足够证明时序预测链路可以真实跑通
- 但它仍是 1.5 的起步版，还需要后续继续做误差解释、分类口径评估与预测结果接页

## 首版评估结果

当前已完成首版评估脚本：

- `scripts/evaluate_fpi_lstm.py`

评估输出：

- `evaluation.json`
- `validation_predictions.csv`
- `loss_curve.png`
- `prediction_scatter.png`
- `label_confusion.png`

当前验证集结果：

- `validation_rows = 234`
- `mae = 0.1629`
- `rmse = 0.2111`
- `r2 = 0.5254`
- `risk_label_accuracy = 0.7821`

当前解读：

- 回归误差已经说明模型学到了部分下一窗口风险变化规律
- 分类准确率说明它在 `low / medium / high` 三档标签上已有初步筛查能力
- 但当前对 `medium` 样本的区分还不够强，后续应重点优化中间等级的识别

## 当前意义

这一步的作用是把 `1.5` 从“方向讨论”推进到“已有真实训练入口”的状态。

后续可以继续做：

1. 训练真实首版 LSTM
2. 评估预测误差与风险等级稳定性
3. 决定是否将预测结果接入单船页或总览页
