# FPI 时序预测模块起步说明

## 当前阶段

这是 `1.5` 的第一步落地版本，目标是先把：

- 时序训练样本
- 轻量模型入口

搭起来，而不是立刻把预测结果接进前端页面。

## 已落地内容

### 1. 序列特征构建模块

文件：

- `scripts/fpi_forecast.py`

作用：

- 将清洗后的 AIS 与环境数据按固定时间窗聚合
- 生成窗口级特征
- 构造监督式序列样本

当前默认输入逻辑：

- `6 小时` 一个窗口
- 取历史 `8` 个窗口作为输入
- 预测下一 `1` 个窗口的 `FPI proxy`

### 2. 数据集导出脚本

文件：

- `scripts/build_fpi_sequence_dataset.py`

作用：

- 直接从当前处理后的 AIS 与环境 CSV 生成时序预测训练数据集

当前已在真实 15 天窗口上验证：

- 窗口行数：`5495`
- 序列样本数：`1160`

### 3. LSTM 训练脚本

文件：

- `scripts/train_fpi_lstm.py`

作用：

- 读取扁平化后的序列数据集
- 按船舶切分训练集与验证集
- 训练首版轻量 `LSTM` 回归模型
- 输出模型权重与训练指标

## 依赖说明

当前仓库默认依赖中还未内置 PyTorch。

原因：

- 1.0 阶段主链路不需要深度学习依赖
- 1.5 当前刚启动，先把数据集和训练入口做好更稳妥

因此：

- `scripts/build_fpi_sequence_dataset.py` 可直接运行
- `scripts/train_fpi_lstm.py` 运行前需要单独安装 PyTorch

## 类别失衡调优结果

在首版评估完成后，我又进一步检查了验证集混淆矩阵，确认当前样本存在明显类别失衡：

- 训练集标签分布约为：
  - `high = 704`
  - `low = 132`
  - `medium = 90`
- 验证集标签分布约为：
  - `high = 175`
  - `low = 40`
  - `medium = 19`

这会直接带来一个问题：模型更容易学会把样本推向 `high`，而对 `medium` 的区分力偏弱。

为此，当前训练脚本已补充两类可配置调优能力：

- `WeightedRandomSampler` 采样平衡
- 加权回归损失

并支持以下参数：

- `--sampling-strategy`
- `--loss-weighting`
- `--sampling-balance-power`
- `--loss-balance-power`

当前已实际比较三套模型：

### 1. 基线模型

- `mae = 0.1629`
- `rmse = 0.2111`
- `r2 = 0.5254`
- `accuracy = 0.7821`

类别召回大致为：

- `low recall = 0.4500`
- `medium recall = 0.2632`
- `high recall = 0.9143`

### 2. 强平衡模型

训练方式：

- `sampling = balanced`
- `loss_weighting = balanced`
- `power = 1.0`

结果：

- `mae = 0.3133`
- `rmse = 0.3432`
- `r2 = -0.2540`
- `accuracy = 0.1838`

类别召回大致为：

- `low recall = 0.7000`
- `medium recall = 0.7895`
- `high recall = 0.0000`

结论：

- 它显著提升了 `medium` 的召回
- 但几乎完全压垮了 `high` 类识别
- 当前不适合作为主模型

### 3. 软平衡模型

训练方式：

- `sampling = balanced`
- `loss_weighting = balanced`
- `power = 0.5`

结果：

- `mae = 0.2113`
- `rmse = 0.2451`
- `r2 = 0.3601`
- `accuracy = 0.7692`

类别召回大致为：

- `low recall = 0.6250`
- `medium recall = 0.2105`
- `high recall = 0.8629`

结论：

- 它比强平衡稳定很多
- 低风险召回有所改善
- 但整体误差和总体准确率仍不如基线
- `medium` 类识别也没有真正解决

## 当前结论

截至当前这轮调优，`baseline model` 仍然是最适合作为 `1.5` 第一版主结果的模型。

原因不是它“完美”，而是它在以下几项之间取得了当前最稳的平衡：

- 回归误差
- 风险等级总体准确率
- `high` 类高风险对象的保持能力

当前建议把这轮调优结果理解为：

- 类别失衡问题已经被明确识别
- 简单的采样平衡和损失加权方向是对的，但当前手段还不够精细
- 下一步更值得尝试的不是继续盲目加大权重，而是：
  1. 多任务学习：同时预测连续 FPI 与离散风险等级
  2. 阈值校准：在输出层做等级边界调优
  3. 特征增强：提高 `medium` 与 `high` 的可分性

## 当前优化与接页结果

在这一轮里，我没有继续盲目重训更多结构，而是先做了两件更有收益的优化：

1. 把训练脚本默认口径修回 `uniform / uniform`
2. 基于验证集预测结果做风险等级阈值校准

原因很简单：

- 当前最好的模型仍然是基线模型
- 如果训练脚本默认继续指向平衡版，后面一旦重跑就会得到更差结果
- 与其继续堆模型复杂度，不如先把“用户实际看到的预测标签”校准到更稳

当前阈值校准结果：

- 默认标签阈值：`0.40 / 0.70`
- 推荐校准阈值：`0.53 / 0.69`

校准后，当前验证集结果变为：

- 原始 `risk_label_accuracy = 0.7821`
- 校准后 `risk_label_accuracy = 0.8120`
- 原始 `macro_f1 = 0.5647`
- 校准后 `macro_f1 = 0.6137`

这意味着：

- 回归模型本身没有改动
- 但前端展示用的风险等级标签口径更稳了
- 这是当前最适合作为单船页预测输出的版本
