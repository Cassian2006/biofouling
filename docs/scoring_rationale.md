# 评分结构说明

## 目标

本次升级将环境打分从“多个变量各自单调加分”的旧结构，调整为：

- `3 个机制层`
- `1 个行为暴露修正层`

目标不是做生态学全真模型，而是让评分结构更符合：

- 热带港区污损形成的真实过程
- IMO 风险管理逻辑
- 平台当前可获得的数据尺度

## 新结构

### 一、机制层

#### 1. 生理适宜层

输入：

- `sst`
- `salinity`

含义：

- 评价该水体是否允许常见附着生物较稳定地存活、生长和繁殖

输出：

- `T = sst_suitability`
- `S = salinity_suitability`

#### 2. 生产力 / 传播压力层

输入：

- `chlorophyll_a`

含义：

- 评价局地食物供给、初级生产力和传播压力是否偏高

输出：

- `P = productivity_pressure_from_chl`

#### 3. 附着水动力层

输入：

- `current_speed = sqrt(current_u^2 + current_v^2)`

含义：

- 评价局地流动条件是否允许幼体靠近表面并尝试临时附着

输出：

- `H = hydrodynamic_attachment_score`

### 二、行为暴露修正层

环境项不再单独主导最终风险，必须与行为暴露联动。

行为暴露当前由以下代理项组成：

- `low_speed_hours`
- `anchor_hours`
- `dwell_hours`
- `port_proximity_hours`
- `port_visits`

## 变量响应函数

### 1. SST

采用 `暖水促进 + 极端高温软回落` 的非对称结构：

- `< 18°C`：明显不利
- `18–25°C`：快速上升
- `25–31°C`：高平台
- `31–33°C`：轻微回落
- `> 33°C`：继续回落，但不直接归零

原因：

- 热带港区更接近长期暖水风险带
- 需要避免“温度越高越高分”的工程化单调假设

### 2. Salinity

采用 `门槛 / 闸门型` 结构：

- `< 20 psu`：强惩罚
- `20–28 psu`：快速上升
- `28–35 psu`：宽平台
- `35–38 psu`：轻微软回落
- `> 38 psu`：继续下降

原因：

- 低盐度会构成明显应激
- 正常海水范围内更像“过滤器”，而不是最大主导变量

### 3. Chlorophyll-a

采用 `压力代理的饱和增益`：

- `< 0.5 μg/L`：低压力
- `0.5–2 μg/L`：中等压力
- `2–8 μg/L`：高压力
- `> 8 μg/L`：封顶，不继续加分

原因：

- 叶绿素更适合表达生产力和传播压力代理
- 目前数据尺度下，封顶比强行回落更稳妥

### 4. Current

不再分别给 `u / v` 打分，而是先合成为：

- `current_speed`

然后按“低流速更利于附着，高流速更不利于附着”建模。

当前切点不直接套实验室流速阈值，而是使用研究区冻结窗口的局地百分位：

- `P25 = 0.3303`
- `P50 = 0.3685`
- `P75 = 0.4189`
- `P90 = 0.4610`

原因：

- 文献决定方向
- 局地分布决定尺度

## 最终合成结构

### 1. 环境修正值

```text
EnvModifier = 0.40T + 0.20S + 0.25P + 0.15H
```

权重逻辑：

- 温度仍是最强直接驱动
- 盐度是闸门，不是主导
- 叶绿素是重要代理，但不高过温度
- 海流是约束项，但受数据尺度限制，不宜过高

### 2. FPI

```text
FPI = BehaviorExposure × (0.7 + 0.3 × EnvModifier)
```

含义：

- 行为暴露仍是主因
- 环境因子只做增强 / 削弱修正
- 避免环境单独把短时高速对象抬成假阳性

### 3. ECP

当前 ECP 仍保留代理惩罚定位，但已改为平滑放大：

- 以 `FPI` 为底
- 由 `maintenance_score`
- `productivity_score`
- `temperature_score`
- `low_speed pressure`
共同放大

注意：

- ECP 仍是工程代理量
- 尚不应解释为真实燃油或真实碳惩罚百分比

### 4. RRI

RRI 延续空间风险结构，但其环境项已切换为新的 `EnvModifier`：

```text
RRI = 0.40 × EnvModifier + 0.25 × Traffic + 0.20 × Anchorage + 0.15 × Behavior
```

## 当前升级的工程策略

- 冻结的竞赛主版本样本保持不被覆盖
- 新科学评分作为优先升级路径与对比层落地
- 旧分数保留为 `legacy` 列，用于对照与答辩说明
