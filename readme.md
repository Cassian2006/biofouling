基于 AIS 与海洋环境的船舶生物污损—隐性碳排—维护决策平台

## 1. 项目简介

本项目面向绿色航运与港口生态安全场景，构建一个基于 **AIS 航迹数据** 与 **海洋环境数据** 的智能分析平台，用于评估船舶在特定航次或特定时间窗内的：

- **生物污损暴露风险**
- **隐性碳排惩罚**
- **维护/清洗决策建议**
- **区域级污损风险图层**

项目不尝试直接测量真实船底污损厚度，也不直接替代工业级船舶能效监测系统，而是构建一个 **“可解释、可视化、可复现”** 的竞赛级数字平台，用于回答：

> 一艘船在什么环境、什么行为模式下，更容易积累船体生物污损？  
> 这些污损可能带来多大的额外能耗与碳排风险？  
> 是否应该清洗？应该何时清洗？在哪类港口环境下风险更高？

## 2. 项目背景

### 2.1 为什么关注生物污损（Biofouling）

船舶在长期航行、低速运行、停泊或锚泊过程中，船体表面会逐渐附着微生物、生物膜、藻类和其他海洋附着生物，这一过程通常被称为 **生物污损**（biofouling）。

生物污损会带来两类核心问题：

1. **航运能效下降与隐性碳排增加**  
   船体附着物会增加航行阻力，导致船舶需要消耗更多燃油维持原有速度，从而提高碳排放。已有研究指出，中等程度的船体污损即可导致显著的温室气体排放增量。

2. **外来入侵水生物种传播风险上升**  
   IMO 已长期将 biofouling 视为船舶传播入侵水生物的重要途径，并在近年持续推进更强的管理框架与在水清洗指导。

因此，生物污损并不是单纯的船舶维护问题，而是一个横跨 **航运能效、碳减排、港口生态安全、绿色治理** 的综合议题。

---

### 2.2 为什么选择新加坡海峡作为研究区域

本项目选取 **新加坡海峡—新加坡港及周边锚地/通道海域** 作为主研究区。原因包括：

- 新加坡是全球最重要的海事枢纽之一，也是全球最繁忙的集装箱中转港之一；官方公开材料显示，每年有超过 13 万艘次船舶到港/挂靠。
- 区域内存在大量 **进出港、锚地等待、低速航行、加油/补给、转运** 等行为，这些行为都与污损积累高度相关。
- 该区域位于热带海域，温暖海水和活跃的生物地球化学条件使其非常适合研究船舶生物污损暴露。
- 环境数据可通过 Copernicus Marine 等公开渠道获得，便于构建可复现的数据流程。

4. 项目目标

本项目的核心目标包括：

4.1 单船/单航次级评估

输入某艘船在指定时间范围内的 AIS 航迹，输出：

航次级污损暴露画像

污损倾向指数（Fouling Propensity Index, FPI）

隐性碳排惩罚估计（Emission Cost Penalty, ECP）

基于大量船舶轨迹和环境数据，输出：

港口/锚地污损暴露等级图

典型高风险航段识别结果

4.3 决策支持

支持用户进行情景对比：

不清洗 vs 近期清洗

不同停泊/等待模式下的风险变化

不同船型或不同运营方式下的隐性碳排差异

7. 核心指标设计（建议版本）

项目先采用“可解释代理指标 + 数据驱动校准”的路线。

7.1 污损倾向指数（FPI）

用于刻画某艘船在特定时间窗口内的污损积累倾向。

建议输入因子：

停泊时长

锚泊时长

低速航行时长

热带高温暴露

高生物活性海域暴露

港口访问频率

连续未维护时长（若可推断）

7.2 隐性碳排惩罚（ECP）

用于估计由于潜在污损引起的额外能耗/碳排风险。

建议采用：

文献经验参数

分级 surrogate 模型

相对惩罚而非绝对工业值

7.3 区域污损风险指数（RRI）

用于生成图层，标识研究区内哪些区域更可能促进船舶污损积累。

建议综合：

环境适宜性

交通密度

低速/停泊概率

港口/锚地暴露强度

第一阶段不要追求全功能，优先实现：

环境数据匹配

只要这个闭环跑通，项目就成立。

原则 2：先做代理指标，再做复杂模型

不要一开始就直接上深度学习。推荐顺序：

规则/统计特征版

简单机器学习版

深度学习增强版

这样更容易验证，也更方便答辩解释。
